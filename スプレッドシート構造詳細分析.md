# スプレッドシート構造詳細分析

## A-Mアカウント（食品コンサル利益）
**URL:** https://docs.google.com/spreadsheets/d/1_Bso-Wf6yGbzIBBSG6vQj_EZ_etcRjKD4G8Fzp1KIAU/edit

### 月別シート構造
**列構成:**
- A列: 注文日
- B列: 商品名
- C列: ASIN
- D列: 販売価格
- E列: 送料
- F列: ポイント
- G列: 割引
- H列: 仕入れ価格
- I列: 手数料
- J列: 資材・送料
- K列: 利益
- L列: TY合計売上

**特徴:**
- 月別シートで管理（01月、02月、03月...）
- 経費管理シートが別途存在
- 不要項目は既に削除済み

## O-AAアカウント（やすくん用_月次収益管理表）
**URL:** https://docs.google.com/spreadsheets/d/1APi6RvBpxSxsNoZCM6Q1cdz9qM_E-5YV2OqVoHdbT08/edit

### 月次収益管理表の構造
**主要項目:**
- カタログ作成数
- 新規カタログリサーチ（ASIN単位）
- 卸への見積もり提案件数（商品数単位）
- 新規取引先開拓「実営業数」
- 実装後は新規開拓の必要なし項目のみ

**KPI管理:**
- 月別の目標値設定
- 実績値の記録
- 進捗管理機能

## システム出力要件の定義

### 必要な出力項目（月次収益管理表形式）

#### 売上高セクション
1. **Amazon（A-Mアカウント）**
   - マカドデータの販売価格合計
   - 送料を含む

2. **Amazon2（O-AAアカウント）**
   - マカドデータの販売価格合計
   - 送料を含む

3. **メルカリShops**
   - メルカリショップデータの売上（税込）合計

4. **バンコ+（楽天等）**
   - 販路プラスデータの楽天売上合計

5. **Yahoo!ショッピング**
   - 販路プラスデータのYahoo売上合計

6. **プリマアプリ**
   - 該当データがある場合

#### 経費セクション
1. **プラットフォーム手数料_Amazon**
   - A-MアカウントのAmazon手数料合計

2. **プラットフォーム手数料_Amazon2**
   - O-AAアカウントのAmazon手数料合計

3. **運送費（送料）**
   - 各販路の送料合計

#### 利益セクション
1. **売上総利益**
   - 全販路の利益合計

### データマッピング

#### マカドデータ → 出力項目
```
販売価格 + 送料 - ポイント - 割引 → Amazon/Amazon2売上高
Amazon手数料 → プラットフォーム手数料
粗利 → 売上総利益に加算
```

#### メルカリショップデータ → 出力項目
```
売上（税込） → メルカリShops売上高
販売手数料（税込） → プラットフォーム手数料に加算
販売利益 → 売上総利益に加算
```

#### 販路プラスデータ → 出力項目
```
mall='rakuten' → バンコ+売上高
mall='yahoo' → Yahoo!ショッピング売上高
mall='mercari' → メルカリShops売上高に加算
profit → 売上総利益に加算
```

## 実装要件

### 1. データ処理ロジック
- 月別集計（注文日/購入日基準）
- アカウント別分離（A-M、O-AA）
- 販路別分類（Amazon、メルカリShops、バンコ+等）

### 2. 出力形式
- 月次収益管理表の項目構造に完全対応
- 日本円形式での表示
- ゼロ値項目の非表示オプション

### 3. 計算精度
- 小数点以下の処理
- 端数処理の統一
- 合計値の整合性確保
